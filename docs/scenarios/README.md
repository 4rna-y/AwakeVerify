# シナリオ別統合仕様書

このディレクトリには、`docs/features` で定義した単体機能を統合し、ユーザーまたはシステムが実際に辿るE2E的な流れをシナリオとして記述する。

## features と scenarios の関係

- `docs/features/*.md`: 単体機能仕様。ログイン、顔認識、眠気スコア算出など、個別機能の入力・処理・出力を定義する。
- `docs/scenarios/*.md`: 統合仕様。複数のfeatureを組み合わせ、ユーザー行動やシステム処理がどの順序で進むかを定義する。

## シナリオ一覧

| ファイル | シナリオ | 概要 |
| --- | --- | --- |
| `student-learning-happy-path.md` | 受講者の通常受講 | 受講者がセッションを開始し、キャリブレーション成功後、通常状態で動画を受講する。 |
| `calibration-retry.md` | キャリブレーション失敗と再試行 | 顔向きや検出状態が不十分でキャリブレーションに失敗し、再試行する。 |
| `drowsiness-auto-pause-resume.md` | 眠気検知による自動停止と再開 | 眠気レベルが危険になり動画が自動停止し、正常復帰後に再開する。 |
| `face-not-detected-warning.md` | 顔未検出時の警告表示 | 顔未検出通知を受け、動画は止めずに警告表示する。 |
| `teacher-dashboard-review.md` | 教員による受講状況確認 | 教員がログインし、受講セッション・眠気スコア・停止イベントを確認する。 |
| `admin-teacher-onboarding.md` | 管理者による教員追加から教員ログインまで | 管理者が教員アカウントを追加し、教員がログイン可能になる。 |

## シナリオ記述方針

各シナリオは以下を含める。

- 目的
- アクター
- 前提条件
- Feature path
- E2Eフロー
- 期待結果
- 例外・分岐
- 関連データ
