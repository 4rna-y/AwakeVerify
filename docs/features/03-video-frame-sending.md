# 映像フレーム送信機能仕様

## 実装優先度

- 優先度: 03
- 理由: Webカメラ映像をバックエンドへ届け、後続の保存・キュー投入・推論処理へ接続するため。

## 1. 機能概要

フロントエンドがWebカメラ映像をエンコード済みI/Pフレームとして、動画再生中にWebSocketでバックエンドへ1フレームずつ送信する機能である。

## 2. 対象コンポーネント

- フロントエンド
- バックエンドWebSocketエンドポイント

## 3. トリガー

受講セッション開始、Webカメラ映像取得、キャリブレーションモーダル完了後、動画再生開始に応じて送信を開始する。

## 4. 接続先

```text
/ws/sessions/{sessionId}/frames
```

## 5. 送信方式

```text
0.0s  I
0.2s  P
0.4s  P
0.6s  P
0.8s  P
1.0s  I
1.2s  P
...
```

仕様:

- 1 WebSocketメッセージ = 1 エンコード済み映像フレーム
- 640×480 / 5fps相当
- Iフレーム間隔は1秒
- Iフレーム間はPフレームのみ
- Bフレームは使用しない
- IフレームはGOP先頭のキーフレームとして扱う

## 6. フレームメタデータ

Iフレーム:

```json
{
  "sessionId": "uuid",
  "sequenceNo": 1,
  "frameType": "I",
  "baseIFrameSequenceNo": 1,
  "capturedAt": "2026-06-14T10:00:00.000Z",
  "codec": "TBD"
}
```

Pフレーム:

```json
{
  "sessionId": "uuid",
  "sequenceNo": 2,
  "frameType": "P",
  "baseIFrameSequenceNo": 1,
  "capturedAt": "2026-06-14T10:00:00.200Z",
  "codec": "TBD"
}
```

## 7. バックエンド受信仕様

バックエンドはフロントエンドからエンコード済み映像フレームを1フレームずつ受信する。

受信時に `receivedAt` を付与する。

## 8. 接続中・リトライ表示

WebSocket接続中は、動画Frame中央にSpinnerを表示し、その下に `接続中` と表示する。

接続に失敗した場合、フロントエンドは指数バックオフで最大5回まで再試行する。

```text
1回目失敗後: 0.5秒後に再試行
2回目失敗後: 1秒後に再試行
3回目失敗後: 2秒後に再試行
4回目失敗後: 4秒後に再試行
5回目失敗後: エラーDialogを表示
```

5回の試行に失敗した場合、動画再生とカメラ画角画像送信は開始せず、エラーDialogでネットワークまたはバックエンド状態の確認を促す。

## 9. 欠落時の前提

Pフレームは順序依存を持つため、欠落または順序不整合が発生した場合、そのGOP内の後続Pフレームは復元不能になる可能性がある。

復旧は次のIフレームで行う。

## 10. 実装設計

現行実装では、1 WebSocketメッセージを以下のJSONとして送信する。

```json
{
  "sessionId": "uuid",
  "sequenceNo": 1,
  "frameType": "I",
  "baseIFrameSequenceNo": 1,
  "capturedAt": "2026-06-14T10:00:00.000Z",
  "codec": "image/jpeg",
  "payloadBase64": "..."
}
```

- `payloadBase64` はエンコード済みフレームバイナリのBase64表現である。
- 現行のブラウザ実装は `image/jpeg` フレームを5fps相当で送信し、`frameType` は1秒周期のGOP境界を表すメタデータとして付与する。
- 将来、差分Pフレーム対応のcodecに置き換える場合も、1メッセージ1フレームとメタデータ項目は維持する。
