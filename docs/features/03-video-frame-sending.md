# 映像フレーム送信機能仕様

## 実装優先度

- 優先度: 03
- 理由: Webカメラ映像をバックエンドへ届け、後続の保存・キュー投入・推論処理へ接続するため。

## 1. 機能概要

フロントエンドがWebカメラ映像を、それぞれ単独でデコード可能な `image/jpeg` 画像フレームとして、動画再生中にWebSocketでバックエンドへ1フレームずつ送信する機能である。

## 2. 対象コンポーネント

- フロントエンド
- バックエンドWebSocketエンドポイント

## 3. トリガー

受講セッション開始、Webカメラ映像取得、キャリブレーションモーダル完了後、動画再生開始に応じて送信を開始する。

## 4. 接続先

```text
/ws/sessions/{sessionId}/frames
```

## 5. 送信方式

仕様:

- 1 WebSocketメッセージ = 1 個別にデコード可能なJPEG画像フレーム
- 640×480 / 5fps相当
- 各フレームは他フレームまたはWorkerのローカル状態に依存しない
- `sequenceNo` はセッション内で単調に増加し、再接続後も継続する

## 6. フレームメタデータ

WebSocketメッセージは次のJSON契約に従う。

```json
{
  "sessionId": "uuid",
  "sequenceNo": 1,
  "capturedAt": "2026-06-14T10:00:00.000Z",
  "videoTimeSec": 123.45,
  "codec": "image/jpeg",
  "payloadBase64": "..."
}
```

- `capturedAt` はUTC時刻とする。
- `videoTimeSec` は動画教材内の再生位置を秒で表す0以上の有限値であり、フレーム番号やFPSから算出しない。
- `payloadBase64` は単独でデコード可能なJPEGバイナリのBase64表現である。

## 7. バックエンド受信仕様

バックエンドはフロントエンドからエンコード済み映像フレームを1フレームずつ受信する。

受信時に `receivedAt` を付与する。

## 8. 接続中・リトライ表示

WebSocket接続中は、動画Frame中央にSpinnerを表示し、その下に `接続中` と表示する。

接続に失敗した場合、フロントエンドは指数バックオフで最大5回まで再試行する。

```text
1回目失敗後: 0.5秒後に再試行
2回目失敗後: 1秒後に再試行
3回目失敗後: 2秒後に再試行
4回目失敗後: 4秒後に再試行
5回目失敗後: エラーDialogを表示
```

5回の試行に失敗した場合、動画再生とカメラ画角画像送信は開始せず、エラーDialogでネットワークまたはバックエンド状態の確認を促す。

接続成功後に予期しない切断が発生した場合も、同じ指数バックオフ（最大5回）で再接続する。再接続中は動画再生とフレーム送信を停止し、再接続できない場合はエラーDialogから再試行できる状態にする。再接続時は送信シーケンス番号をセッション内で継続し、古い接続のタイマーやフレーム送信を残さない。

## 9. 欠落時の前提

フレームの欠落または順序不整合があっても、後続の有効なJPEGフレームは個別に処理できる。受信・処理側は欠落を理由に後続フレームを破棄しない。

## 10. 実装設計

現行実装は、上記のJSON契約で `image/jpeg` フレームを5fps相当で送信する。フレーム種別またはフレーム間参照を表すメタデータは送信しない。
