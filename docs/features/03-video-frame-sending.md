# 映像フレーム送信機能仕様

## 実装優先度

- 優先度: 03
- 理由: Webカメラ映像をバックエンドへ届け、後続の保存・キュー投入・推論処理へ接続するため。

## 1. 機能概要

フロントエンドがWebカメラ映像をエンコード済みI/Pフレームとして、WebSocketでバックエンドへ1フレームずつ送信する機能である。

## 2. 対象コンポーネント

- フロントエンド
- バックエンドWebSocketエンドポイント

## 3. トリガー

受講セッション開始後、Webカメラ映像の取得が可能になった時点で送信を開始する。

## 4. 接続先

```text
/ws/sessions/{sessionId}/frames
```

## 5. 送信方式

```text
0.0s  I
0.2s  P
0.4s  P
0.6s  P
0.8s  P
1.0s  I
1.2s  P
...
```

仕様:

- 1 WebSocketメッセージ = 1 エンコード済み映像フレーム
- 640×480 / 5fps相当
- Iフレーム間隔は1秒
- Iフレーム間はPフレームのみ
- Bフレームは使用しない
- IフレームはGOP先頭のキーフレームとして扱う

## 6. フレームメタデータ

Iフレーム:

```json
{
  "sessionId": "uuid",
  "sequenceNo": 1,
  "frameType": "I",
  "baseIFrameSequenceNo": 1,
  "capturedAt": "2026-06-14T10:00:00.000Z",
  "codec": "TBD"
}
```

Pフレーム:

```json
{
  "sessionId": "uuid",
  "sequenceNo": 2,
  "frameType": "P",
  "baseIFrameSequenceNo": 1,
  "capturedAt": "2026-06-14T10:00:00.200Z",
  "codec": "TBD"
}
```

## 7. バックエンド受信仕様

バックエンドはフロントエンドからエンコード済み映像フレームを1フレームずつ受信する。

受信時に `receivedAt` を付与する。

## 8. 欠落時の前提

Pフレームは順序依存を持つため、欠落または順序不整合が発生した場合、そのGOP内の後続Pフレームは復元不能になる可能性がある。

復旧は次のIフレームで行う。
