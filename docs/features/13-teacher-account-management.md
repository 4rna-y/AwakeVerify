# 教員アカウント管理機能仕様

## 実装優先度

- 優先度: 13
- 理由: 教員利用者を管理し、ログイン可能な教員アカウントを追加するため。

## 1. 機能概要

管理者が教員アカウントを追加する機能である。

管理者ページから教員IDとパスワードを入力し、バックエンドでパスワードをハッシュ化して `teachers` に保存する。

## 2. 利用者

- 管理者

## 3. 対象コンポーネント

- フロントエンド
- バックエンド
- PostgreSQL

## 4. トリガー

管理者が `/admin/teachers` 画面で教員アカウント追加操作を行う。

## 5. 入力

| 項目 | 必須 | 入力制約 | 表示 |
| --- | --- | --- | --- |
| 教員ID | 必須 | 空文字不可。前後空白は送信前に除去する。 | `Input` |
| パスワード | 必須 | 空文字不可。入力内容は画面上でマスクする。 | `Input type="password"` |

パスワードはフロントエンド上で平文保存しない。

## 6. API仕様

```http
POST /api/admin/teachers
```

request:

```json
{
  "teacherId": "string",
  "password": "string"
}
```

## 7. 処理仕様

### 7.1 フロントエンド

1. 教員IDとパスワード入力フォームを表示する。
2. 入力内容を検証する。
3. バックエンドへ送信する。
4. 送信成功時は教員一覧を更新する。
5. 送信失敗時は `Alert` でエラーを表示する。
6. 入力内容をフロントエンド永続領域へ保存しない。

### 7.2 バックエンド

1. 管理者認証を確認する。
2. `teacherId` の重複を確認する。
3. パスワードをハッシュ化する。
4. `teachers` に保存する。

## 8. データ保存

```text
teachers
- teacher_id
- password_hash
- created_at
- created_by_admin_id nullable

admins
- admin_id
- password_hash
- created_at
```

管理者自身もIDとパスワードで認証される前提とする。
