# Webカメラ取得機能仕様

## 実装優先度

- 優先度: 02
- 理由: キャリブレーション、顔認識、映像フレーム送信の入力となるWebカメラ映像を取得するため。

## 1. 機能概要

受講者のWebカメラ映像を取得し、キャリブレーション、顔検出、眠気判定に使用する映像入力を準備する機能である。

## 2. 利用者

- 受講者

## 3. 対象コンポーネント

- フロントエンド
- ブラウザMediaStream API

## 4. トリガー

受講者セッション開始後、フロントエンドがWebカメラ使用許可を要求する。

## 5. 入力

Webカメラから以下の条件で映像を取得する。

```text
解像度: 640×480
フレームレート: 5fps相当
```

## 6. 処理仕様

1. 受講者にWebカメラ使用許可を求める。
2. 許可後、カメラ映像を取得する。
3. カメラプレビューを表示する。
4. 取得した映像をキャリブレーションおよびWebSocket映像送信に使用する。

## 7. 画面仕様

カメラプレビューは受講者が姿勢を確認するために表示する。

ただし、動画教材より小さく表示する。

受講者画面では以下の領域に表示する。

| 領域 | 内容 | 配置・大きさの方針 |
| --- | --- | --- |
| サイド / 下部 | カメラプレビュー、キャリブレーション進捗、眠気レベル、警告 | デスクトップでは動画右側、狭幅画面では動画下に配置する。 |

## 8. エラー・例外

- カメラ権限未取得時は、カメラ許可案内を表示する。
- 復旧不能な権限エラー時は、エラーメッセージと再試行導線を表示する。
