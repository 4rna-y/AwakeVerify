# Webカメラ取得機能仕様

## 実装優先度

- 優先度: 02
- 理由: キャリブレーション、顔認識、映像フレーム送信の入力となるWebカメラ映像を取得するため。

## 1. 機能概要

受講者のWebカメラ映像を取得し、キャリブレーション、顔検出、眠気判定に使用する映像入力を準備する機能である。

## 2. 利用者

- 受講者

## 3. 対象コンポーネント

- フロントエンド
- ブラウザMediaStream API

## 4. トリガー

受講者セッション開始後、フロントエンドがWebカメラ使用許可を要求する。

## 5. 入力

Webカメラから以下の条件で映像を取得する。

```text
解像度: 640×480
フレームレート: 5fps相当
```

## 6. 処理仕様

1. Loginページから `/student/session` の動画再生ページへ遷移する。
2. 受講者にWebカメラ使用許可を求める。
3. 許可後、カメラ映像を取得する。
4. 動画Frame上のキャリブレーションモーダルにカメラ画角を表示する。
5. 取得した映像をキャリブレーションおよびWebSocket映像送信に使用する。

## 7. 画面仕様

カメラプレビューは受講者が姿勢を確認するために表示する。

ただし、動画教材より小さく表示する。

受講者画面では以下の領域に表示する。

| 領域 | 内容 | 配置・大きさの方針 |
| --- | --- | --- |
| 動画Frame上のモーダル | キャリブレーション用カメラ画角、5秒進捗、開始ボタン | `/student/session` 遷移後に表示し、完了後に閉じる。 |
| 動画Frame右上 | カメラ録画中表示 | 動画Frame上でFloat表示する。 |

## 8. エラー・例外

- カメラ権限未取得時は、カメラ許可案内を表示する。
- 復旧不能な権限エラー時は、エラーメッセージと再試行導線を表示する。
